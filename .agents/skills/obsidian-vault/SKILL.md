# Obsidian Vault

## Vault location

`~/notes` (= `/home/fomar/notes`).

PARA under `docs/`: `00-projects`, `10-areas`, `20-resources`, `30-dailynotes`, `50-templates`, `90-archives`, `plannotator/`.
Root dirs: `cards/` (hashcards spaced-repetition store — not notes), `handoff/`, `jobs/`, `scripts/`, `tmp/`, `ssh/`. `90-archive/prompts/` holds `ocp` prompt history — not notes.
Loose root `.md` notes are Inbox — file them by PARA at weekly review.

## Naming & links

- **Filenames**: lowercase-kebab-case (`note_id_func` lowercases, spaces→hyphens, strips non-alnum).
- **PARA prefixes**: 2-digit zero-padded sequential (`02-`, `04-`, …). Pick next free number, don't reuse. Preserve when moving project↔archive.
- **Wiki links**: target by **kebab-case id** (Obsidian resolves by basename). Hand-written forms that resolve: `[[kebab-id]]` or `[[relative/path/kebab-id|Display]]`. obsidian.nvim's alias-only `use_alias_only` is fine; its path-prefix `format_link` breaks rendering — prefer hand-built path links for nested notes.
- **Frontmatter `id`**: matches the kebab-case filename. `:ObsidianNew` sets it; created via a file tool, add `id: <kebab-filename>` yourself.
- **Frontmatter off**: `tasks/` dirs and `kanban.md` (kanban.nvim).

## Daily notes

Path `docs/30-dailynotes/YYYY/MM/YYYY-MM-DD.md`; skeleton from `docs/50-templates/dailynote-template.md`.
Sections: Summary → Journal → Tangent Parking Lot → Operations → To-Do → Completed → Canceled → Notepad → Log.
Log metadata: `**Days Sober:**`, `**Hours of Sleep:**`, `**Celibacy Countdown:**`, `- **HH:MM** **fN/10** **mN/5**`.

## Weekly notes

Path `docs/30-dailynotes/weeklynotes/YYYY/YYYY-Wnn.md`. **Programmatic — never hand-write**: generated by `custom/weekly-note.lua` via `:ObsidianWeekly [YYYY-MM-DD]` / `:ObsidianWeeklyPrev` / `:ObsidianWeeklyNext`. `[[YYYY-Wnn]]` links auto-create via `follow_weekly_link`.
Sections: Health dashboard, Startup, Week at a Glance, Inbox, Tangent Parking Lot, Summary Digest, End Of Week Review, Planned Tasks (carries `### Goal:` from previous week).

## Tasks

States: `[ ]` todo, `[/]` in-progress, `[x]` done, `[-]` cancelled.
Priority emoji: 🔺 critical · ⏫ high · 🔼 medium · 🔽 low · ⏬ very low.
Progress: `- [0/3] task text 🔺`.

`task-auto-complete.lua` relocates `- [x]` tasks into `## Completed` (above `## Canceled`) at the bottom of the file on `BufWritePost *.md`. **To complete a task: check the box and save — the plugin moves it. Never relocate by hand.**

`:ObsidianTasksByTag [tag1] [tag2]` — filters tasks by file-level tags (AND over frontmatter + inline); incomplete by default.

Kanban (`kanban.nvim`): `./tasks/` folder, `## ` list heads.

## Workflows

### Find notes

```bash
rg -l "keyword" ~/notes --type md            # content search
rg -l '\[\[kebab-id\]\]' ~/notes --type md    # backlinks to a kebab-id
```

### Create a note

1. Pick a lowercase-kebab-case filename (rules in Naming).
2. File by PARA: project → `docs/00-projects/<NN>-<name>-project.md` (`-project` suffix is conventional but not universal — e.g. `23-blog`), topic → `docs/20-resources/`, person → `docs/10-areas/people/<name>.md`.
3. Copy the matching template skeleton (dailies/weeklies are generated — no template).
4. Add `[[wikilinks]]` (kebab-id form) to related notes.
5. Set frontmatter `id` if using a file tool (Naming).

Done when: note is in its PARA folder, frontmatter `id` is set, and every related note that already exists is linked.

### Move to archive

Send stale projects AND loose notes to `docs/90-archives/`, preserving the numbered prefix (rules in Naming). `git mv`, then verify backlinks — obsidian.nvim rename handles ref updates; confirm after.
Done when: file relocated, prefix preserved, and every backlink resolves.

### Edit a daily note

Follow the template skeleton. To complete a task: check `- [x]` and save (Tasks — the plugin relocates it).

### Create a weekly note

Run `:ObsidianWeekly` in nvim (Weekly notes).
